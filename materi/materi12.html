APA ITU SQL, DDL, DML, DCL, dan TCL?

Apa itu SQL?

SQL (Structured Query Language) adalah bahasa standar yang digunakan untuk mengakses, mengelola, dan memanipulasi data di dalam database relasional. SQL memungkinkan kita untuk membuat database, mengisi data, menampilkan data, memperbarui data, dan juga menghapus data.


SQL pertama kali dikembangkan oleh IBM pada tahun 1970-an dan kini menjadi bahasa universal yang digunakan di hampir semua sistem database.


Fungsi Utama SQL:

  • Membuat struktur database (tabel, kolom, relasi).
  • Memasukkan, menampilkan, atau memperbarui data.
  • Mengatur hak akses pengguna.
  • Melakukan analisis dan pencarian data.

SQL Commands



DDL (Data Definition Language)

DDL (Data Definition Language) adalah bagian dari SQL yang digunakan untuk mendefinisikan struktur database. Dengan DDL kita bisa membuat tabel baru, mengubah struktur tabel yang sudah ada, atau menghapus tabel yang tidak diperlukan.


Perintah DDL biasanya langsung disimpan secara permanen di database, artinya tidak bisa dibatalkan dengan "ROLLBACK".


Fungsi Utama DDL:

  • CREATE : Mmebuat tabel atau database baru.
  • ALTER : Mengubah struktur tabel (misalnya menambah kolom baru).
  • DROP : Menghapus tabel atau database
  • TRUNCTAE : Menghapus semua isi tabel tanpa menghapus strukturnya.

DML (Data Manipulation Language)

DML (Data Manipulation Language) digunakan untuk memanipulasi isi data yang terdapat di dalam tabel. Dengan DML, kita bisa menambah, menampilkan, mengubah, atau menghapus data tanpa mengubah struktur tabelnya.


Berbeda dengan DDL, Perintah DML bisa dibatalkan menggunakan TCL (Transaction Control Language) seperti "ROLLBACK" jika perubahan belum disimpan dengan "COMMIT".


Fungsi Utama DML:

  • INSERT : Menambahkan data baru ke tabel.
  • SELECT : menampilkan data dari tabel.
  • UPDATE : Mengubah data yang sudah ada.
  • DELETE : Menghapus data dari tabel

DCL (Data Control Language)

DCL (Data Control Language) digunakan untuk mengatur hak akses dan izin pengguna dalam database. Dengan DCL, administrator database dapat memberikan atau mencabut hak tertentu kepada pengguna agar sistem tetap aman dan terkontrol.


Fungsi Utama DCL:

  • GRANT : Memberikan izin (hak akses) kepada pengguna.
  • REVOKE : Mencabut izin yang telah diberikan.

TCL (Transaction Control Language)

TCL (Transaction Control Language) adalah bagian dari SQL yang digunakan untuk mengatur proses transaksi di dalam database. Transaksi dalam database berarti serangkaian perintah yang dijalankan secara bersama-sama, jika salah satu gagal, maka semuanya bisa dibatalkan agar tetap konsisten.


TCL biasanya digunakan bersama dengan perintah DML seperti INSERT, UPDATE, dan DELETE.


Fungsi Utama TCL:

  • COMMIT : Menyimpan semua perubahan yang telah dilakukan.
  • ROLLBACK : Membatalkan perubahan jika terjadi kesalahan.
  • SAVEPOINT : Menandai titik tertentu dalam transaksi agar bisa kembali ke titik itu jika perlu.

Kesimpulan

SQL merupakan bahasa utama dalam dunia basis data, dan di dalamnya terdapat lima bagian penting yang memiliki fungsi berbeda-beda:

  • DDL untuk membuat dan mengubah struktur database.
  • DML untuk mengelola isi data di dalam tabel.
  • DCL untuk mengatur hak akses pengguna.
  • TCL untuk menjaga keutuhan data dalam transaksi.

Mempelajari perintah-perintah ini adalah langkah pertama untuk bekerja secara efektif dengan data. Semakin Anda menggunakannya, semakin mudah untuk mendapatkan database untuk melakukan persis apa yang Anda butuhkan.
