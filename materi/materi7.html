APA ITU PRIMARY KEY, FOREIGN KEY, DAN CANDIDATE KEY BESERTA CONTOHNYA

APA ITU KEY DALAM ERD?

Key dalam Entity Relationship Diagram (ERD) adalah atribut yang dipilih untun mengidentifikasi entitas secara unik. Keunikan atribut key memungkinkan entitas tersebut dibedakan dari entitas lain dalam basis data.


APA FUNGSI KEY?

Key dalam ERD berfungsi untuk memberikan identifikasi pada setiap baris data dalam sebuah tabel dan sebagai penghubung antara satu tabel dan juuga dengan tabel lainnya. Key juga sangat penting sebagai tanda pengenal dan penghubung antar tabel dalam suatu relational database. Key dapat dipahami sebagai 'tanda pengenal' untuk mengidentifikasi sebuah baris di dalam tabel. Key juuga berperan penting sebagai penghubung antara satu tabel dan juga dengan tabel lainnya. Key dalam database juga berfungsi untuk menjalankan fungsi sebagai 'tanda pengenal' dan sebagai penghubung antar tabel.


PENGERTIAN PRIMARY KEY

Primary Key adalah atribut atau kumpulan atribut yang digunakan untuk mengidentifikasi setiap baris data dalam sebuah tabel secara unik. Primary Key dipilih karena keunikan, sehingga dapat dibedakan dengan baris data yang lain. Primary Key berfungsi sebagai tanda pengenal dan penghubung antar tabel dalam satu relational database. Primary Key juga sangat penting dalam proses pencarian data dan memastikan integritas antar data pada tabel.


PERAN DAN PENTINGNYA PRIMARY KEY

Peran dan pentingnya Primary Key dalam ERD antara lain:

  1. Memberikan identifikasi pada setiap baris data dalam sebuah tabel
  2. Menjadi penghubung antara satu tabel dan juga dengan tabel lainnya.
  3. Memastikan setiap baris data dalam tabel bersifat unik atau berbeda antara baris satu dengan baris lain.
  4. Memudahkan dalam pengolahan data dan proses pencarian data.
  5. Menjaga integritas antar data pada tabel.
  6. Mencegah terjadinya duplikasi data.
  7. Membantu melakukan pencarian dengan waktu yang cepat.

Primary Key dapat dipilih dari satu atau kumpulan atribut yang memenuhi syarat-syarat tertentu, seperti harus berisi nilai unik, tidak boleh berisi nilai kosong, dan harus berdiri sendiri pada tabel atau atomik. Primary Key juga harus dipilih dengan hati-hati agar tidak terjadi banyak perubahan karena kesalahan.


Dalam ERD, Primary Key direpresentasikan dengan simbol kunci atau underline pada atribut yang dipilih sebagai Primary Key. Primary Key juga dapat digunakan sebagai Foreign Key pada tabel lain untuk menghubungkan antar tabel.

PENGERTIAN FOREIGN KEY

Foreign Key adalah atribut atau kumpulan atribut yang digunakan untuk menghubungkan antara tabel master dengan tabel turunannya. Foreign Key digunakan untuk menciptakan hubungan antara dua tabel dalam database relasional. Foreign Key juga dikenal sebagai kunci tamu karena merupakan kunci yang terdapat pada tabel kedua atau child. Foreign Key merujuk pada suatu kolom yang unik untuk menghubungkan antar tabel.


PERAN DAN PENTINGNYA FOREIGN KEY

Peran dan pentingnya Foreign Key dalam ERD antara lain

  1. Menghubungkan antara tabel master dengan tabel turunannya.
  2. Menciptakan hubungan antara dua tabel dalam database relasional.
  3. Memastikan integritas referensial antar tabel.
  4. Memudahkan dalam pengolahan data dan proses pencarian data.
  5. Menjaga integritas antar data pada tabel.
  6. Mencegah terjadinya duplikasi data.

Foreign Key harus dipilih dengan hati-hati agar tidak terjadi banyak perubahan karena kesalahan. Foreign Key juga harus memenuhi syarat-syarat tertentu, seperti harus berisi nilai unik, tidak boleh berisi nilai kosong, dan harus berdiri sendiri pada tabel atau atomik.


Dalam ERD, Foreign Key direpresentasikan dengan simbol kunci yang terhubung dengan Primary Key pada tabel lain. Foreign Key juga dapat digunakan sebagai Primary Key pada tabel turunan untuk menghubungkan antar tabel.


CONTOH PENERAPAN FOREIGN KEY DALAM ERD

  1. Hubungan antara Tabel "Mahasiswa" dan "Wali Dosen

    2. Dalam ERD sistem akademik ini, tabel "MAHASISWA" memiliki Foreign Key dalam bentuk atribut "WALI DOSEN_nidn" yang merujuk ke Primary Key "nidn" dalam tabel "WALI DOSEN." Hal ini memungkinkan kita untuk mengidentifikasi dosen wali yang bertanggung jawab atas mahasiswa tertentu.

  2. Hubungan antara Tabel "Sales" dan "Pelanggan"

    3. Dalam ERD sistem penjualan ini, tabel "PELANGGAN" memiliki Foreign Key dalam bentuk atribut "SALES_id_sales" yang merujuk ke Primary Key "id_sales" dalam tabel "SALES." Hal ini memungkinkan kita untuk mengidentifikasi petugas sales yang melayani atau bertanggung jawab atas pelanggan tertentu.

  3. Hubungan antara Tabel "Kategori" dan "Produk"

    4. Dalam ERD sistem penjualan ini, tabel "PRODUK" memiliki Foreign Key dalam bentuk atribut "KATEGORI_id_kategori" yang merujuk ke Primary Key "id_kategori" dalam tabel "KATEGORI." Hal ini memungkinkan kita untuk mengidentifikasi kategori yang menjadi kelompok produk tertentu.

  4. Hubungan antara Tabel "Hotel" dan "Kamar"

    5. Dalam ERD sistem reservasi hotel ini, tabel "KAMAR" memiliki Foreign Key dalam bentuk atribut "HOTEL_id_hotel" yang merujuk ke Primary Key "id_hotel" dalam tabel "HOTEL." Hal ini memungkinkan kita untuk mengidentifikasi hotel tempat kamar tertentu berada.

  5. Hubungan antara Tabel "Supplier" dan "Barang"

    6. Dalam ERD sistem inventaris atau penjualan ini, tabel "BARANG" memiliki Foreign Key dalam bentuk atribut "SUPPLIER_id_supplier" yang merujuk ke Primary Key "id_supplier" dalam tabel "SUPPLIER." Hal ini memungkinkan kita untuk mengidentifikasi pemasok (supplier) yang menyediakan barang tertentu.


PENGERTIAN CANDIDATE KEY

Candidate Key adalah kumpulan atribut yang dapat membedakan setiap baris data dalam sebuah relasi secara unik. Candidate Key juga dikenal sebagai kunci calon karena merupakan kunci yang memenuhi syarat untuk menjadi Primary Key. Candidate Key berfungsi sebagai tanda pengenal dan penghubung antar tabel dalam suatu relational database.


PERAN DAN PENTINGNYA CANDIDATE KEY

Peran dan pentingnya Candidate Key dalam ERD antara lain:

  1. Memberikan identifikasi pada setiap baris data dalam sebuah tabel.
  2. Menjadi kandidat untuk menjadi Primary Key pada tabel.
  3. Memastikan setiap baris data dalam tabel bersifat unik atau berbeda antara baris satu dengan baris lain.
  4. Memudahkan dalam pengolahan data dan proses pencarian data.
  5. Menjaga integritas antar data pada tabel.
  6. Mencegah terjadinya duplikasi data.

Candidate Key harus dipilih dengan hati-hati agar tidak terjadi banyak perubahan karena kesalahan. Candidate Key juga harus memenuhi syarat-syarat tertentu, seperti harus berisi nilai unik, tidak boleh berisi nilai kosong, dan harus berdiri sendiri pada tabel atau atomik.


Dalam ERD, Candidate Key direpresentasikan dengan simbol underline pada atribut yang memenuhi syarat untuk menjadi Primary Key. Candidate Key juga dapat digunakan sebagai Foreign Key pada tabel lain untuk menghubungkan antar tabel.


CONTOH PENERAPAN CANDIDAT KEY DALAM ERD

  1. Tabel "Mahasiswa"

    2. Dalam ERD sistem akademik, tabel "MAHASISWA" memiliki minimal dua Candidate Key, yaitu "nim" (Nomor Induk Mahasiswa) dan "email". Kedua atribut ini memenuhi syarat sebagai kunci unik yang dapat digunakan untuk mengidentifikasi setiap mahasiswa secara unik.

  2. Tabel "Wali Dosen"

    3. Dalam ERD sistem akademik, tabel "WALI DOSEN" memiliki minimal dua Candidate Key, yaitu "nidn" (Nomor Induk Dosen Nasional) dan "email". Kedua atribut ini dapat digunakan untuk mengidentifikasi setiap dosen wali secara unik.

  3. Tabel "Pelanggan"

    4. Dalam ERD sistem penjualan, tabel "PELANGGAN" memiliki minimal dua Candidate Key, yaitu "id_pelanggan" dan "email". Kedua atribut ini dapat digunakan untuk mengidentifikasi setiap pelanggan secara unik.

  4. Tabel "Sales"

    5. Dalam ERD sistem penjualan, tabel "SALES" memiliki minimal dua Candidate Key, yaitu "id_sales" dan "email_sales". Kedua atribut ini dapat digunakan untuk mengidentifikasi setiap petugas sales secara unik.

  5. Tabel "Produk"

    6. Dalam ERD toko online, tabel "PRODUK" memiliki minimal dua Candidate Key, yaitu "id_produk" dan "nama_produk". Kedua atribut ini mungkin dapat digunakan untuk mengidentifikasi setiap produk secara unik, asumsi nama produk tidak pernah berulang.
